## WORKFLOW — HOW CALENDAR CONTENT REQUESTS FLOW

```
1. Authority Engine™ outputs campaign brief (niche, avatar, quarter, belief stage)
2. Sloane Mercer receives brief → confirms inputs → invokes Strategy Coordinator
3. Nadia Voss: queries Airtable → selects monthly themes → outputs Theme Brief
4. Sloane Mercer: stores Theme Brief → initiates weekly cycles
5. Caleb Navarro: queries Airtable → selects weekly event + daily hooks → outputs Content Calendar Block
6. Lena Ashby: receives Content Calendar Block → produces content pieces (social, email, YouTube outline, long-form)
7. Sloane Mercer: packages Weekly Draft Package → presents to Roger for review
8. Roger: approves or returns with revision notes
9. If revision: Sloane Mercer routes to appropriate team member based on revision type
10. Approved content → Publishing Layer
```

---

## COLLABORATION PATTERNS

- **Sloane → All**: Sloane initiates every handoff and manages all context state. No team member begins work without a Sloane-confirmed brief.
- **Nadia → Caleb**: Theme Brief is the primary input for Content Calendar Block construction. Caleb cannot begin Step 5 without a completed Theme Brief.
- **Caleb → Lena**: Content Calendar Block is the primary input for content production. Lena cannot begin writing without a completed Block.
- **Lena → Sloane**: All draft output returns to Sloane for packaging before Roger review.
- **Roger → Sloane**: All revision notes route through Sloane, who determines revision type and downstream routing.
