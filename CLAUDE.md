## NON-NEGOTIABLES

1. **Never generic.** Every output must reference the client's specific niche, geography, named services, ideal avatar, and competitive context.

2. **The client is the prize.** If it doesn't move beliefs, pipeline, or revenue, it doesn't belong in the deliverable.

3. **Belief-to-Buy Framework™ is the underlying architecture.** Dual-track parallel system — both tracks must reach alignment simultaneously before a buying decision occurs.

   BELIEF TRACK: Enemy Belief → Pre-Frame → New Worldview → Internal Alignment → Certainty
   EMOTION TRACK: Dissatisfaction → Contrast → Desire → Urgency → Relief

   Both converge at BUY. Moving only one track stalls the prospect. This is the Authority Conversion Protocol™.
   Every content piece, email, ad, and video maps to a specific stage pair on BOTH tracks.
   See `/skills/strategy/BELIEF_FILTER_MAP.md`.

4. **Roger's voice is the standard.** Direct. High conviction. Speaks in outcomes, not tactics. See `/brand-standards/ROGER_VOICE.md`.

5. **Quality gates are mandatory.** Every deliverable passes through all four QC gates. See `/brand-standards/QUALITY_CHECKLIST.md`.

   **Attorney clients — mandatory additional gate:** All content (service pages, about pages, emails, landing pages, social, direct mail) must pass through the ABA Compliance Checker before delivery. See `/skills/compliance/ABA_COMPLIANCE_CHECKER.md`. Runs after content is written, before DOCX is built. No attorney deliverable ships without it.

6. **The team is real to the client.** Every strategic document reflects the appropriate team member's lens and sign-off.

7. **Content Sync Check — conditional pre-flight:**

   - **Recurring content** (LinkedIn posts, email sequences, calendar pieces, social): Always run `/skills/calendar/CONTENT_SYNC_CHECKER.md` first. Pass the Sync Context Block to the content specialist before they begin.
   - **One-time documents** (service pages, about pages, FAQs, proposals, blueprints, case studies): Skip sync check unless Roger flags otherwise.
   - When Roger designates a request "standalone," skip unconditionally.

- Table of contents for documents over 5 pages
- All sections complete — no placeholder text
- Client-specific language throughout
- All four QC gate sign-offs

**File naming:** `[client-slug]_[deliverable-name]_[YYYYMMDD].[ext]`

**Output folder structure (mandatory):**

```text
/client-onboarding/clients/[client-slug]/outputs/

  docx/   ← Word documents
  pdf/    ← PDF reports
```

Build scripts (`.py`) → `outputs/` root. After saving, log in `deliverable-log.md` immediately.

If the client folder does not yet exist, create the full structure (client-context.yaml stub, deliverable-log.md, outputs/md/, outputs/docx/, outputs/pdf/) before saving any file.

The ASG root (`/`) is for `CLAUDE.md` and `CLAUDE_REFERENCE.md` only. The `/outputs/` directory is for ASG-internal assets only — never client deliverables.
